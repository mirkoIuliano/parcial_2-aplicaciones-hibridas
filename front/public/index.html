Rutas para usar el CRUD de Usuarios

Método Función URL
GET Ver todos los usuarios http://127.0.0.1:3000/api/usuarios
GET Ver un usuario especifico
Es necesario poner como parámetro el id del usuario que se desea buscar		 http://127.0.0.1:3000/api/usuarios/:id
POST Para crear un usuario http://127.0.0.1:3000/api/usuarios
PUT Para modificar un usuario existente
Es necesario poner como parámetro el id del usuario que se desea modificar		 http://127.0.0.1:3000/api/usuarios/:id
DELETE Para modificar un usuario existente
Es necesario poner como parámetro el id del usuario que se desea borrar		 http://127.0.0.1:3000/api/usuarios/:id
POST Para iniciar sesión
Requiere email y password		 http://127.0.0.1:3000/api/usuarios/iniciar-sesion

Notas adicionales

Los parámetros del modelo User son: name, email y password

Ejemplo de objeto JSON:

{
"name": "Nombre de usuario", // Mínimo 3 caracteres
"email": "test@gmail.com", // No se puede poner un email que ya está en uso
"password": "contraseña" // Mínimo 8 caracteres
}

Rutas para usar el CRUD de Alumnos

Método Función URL
GET Ver todos los alumnos http://127.0.0.1:3000/api/alumnos
GET Ver un alumno especifico
Es necesario poner como parámetro el id del alumno que se desea buscar		 http://127.0.0.1:3000/api/alumnos/:id
POST Para crear un alumno http://127.0.0.1:3000/api/alumnos
PUT Para modificar un alumno existente
Es necesario poner como parámetro el id del alumno que se desea modificar		 http://127.0.0.1:3000/api/alumnos/:id
DELETE Para modificar un alumno existente
Es necesario poner como parámetro el id del alumno que se desea borrar		 http://127.0.0.1:3000/api/alumnos/:id
GET Ver un alumno especifico mediante su nombre
Es necesario poner como parámetro el nombre del alumno que se desea buscar, teniendo en cuenta mayúsculas y minúsculas		 http://127.0.0.1:3000/api/alumnos/buscar/:name
GET

Filtro

Filtra por año y muestra todos los alumnos que pertenecen a ese año
Es necesario poner como parámetro el año de cursada que se desea filtrar (de primer a sexto año)		 http://127.0.0.1:3000/api/alumnos/año/:school_year

Notas adicionales

Los parámetros del modelo Student son: name, last_name, school_year y birthdate

Ejemplo de objeto JSON:

{
"name": "Nombre de alumno", // Mínimo 2 caracteres
"last_name": "Apellido de alumno", // Mínimo 2 caracteres
"school_year": 1, // Se escribe con número y va del 1 al 6 (primer año a sexto año)
"birthdate": "2000-12-31" // El formato es: YYYY-MM-DD
}

Rutas para usar el CRUD de Materias

Método Función URL
GET Ver todos los materias http://127.0.0.1:3000/api/materias
GET Ver las materias que tiene un alumno
Es necesario poner como parámetro el id del alumno		 http://127.0.0.1:3000/api/materias/:id
POST Para crear una materia http://127.0.0.1:3000/api/materias
PUT Para modificar una materia existente
Es necesario poner como parámetro el id de la materia que se desea modificar		 http://127.0.0.1:3000/api/materias/:id
DELETE Para modificar una materia existente
Es necesario poner como parámetro el id de la materia que se desea borrar		 http://127.0.0.1:3000/api/materias/:id
GET

Filtro

Filtra por materia y muestra todos los alumnos que están en esa materia
Es necesario poner como parámetro la materia que se desea filtrar (con mayúsculas, minúsuclas y acentos)		 http://127.0.0.1:3000/api/materias/materia/:subject

Notas adicionales

Los parámetros del modelo Subject son: name y studentId

Ejemplo de objeto JSON:

{
"name": "Nombre de la materia",
"studentId": "670ae395f783cfbf29096fb3" // Aquí iría el ID del alumno al cual se está haciendo referencia
}
